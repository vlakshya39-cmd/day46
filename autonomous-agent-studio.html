Extension Ideas

      

        
    
          
  • Add a Test Generator agent that writes pytest cases before the Executor runs, and feed pass/fail counts into the Evaluator's rubric.
    • 
          
  • Swap the single-function scope for multi-file projects, with a Planner that outputs a file tree and an Executor that streams one file at a time.
    • 
          
  • Persist Memory Manager notes across sessions (localStorage or a backend) so recurring problem types start from prior learnings instead of a cold start.
    • 
          
  • Add a human-in-the-loop gate: pause after Final Reviewer and let a person approve, reject, or request one more Improver pass.
    • 
          
  • Parallelize independent sub-tasks (e.g. multiple candidate implementations) and have the Evaluator pick the strongest before the Critic/Improver loop begins.
    • 
          
  • Replace the fixed rubric with a dynamically generated one: have the Planner draft evaluation criteria specific to the problem before Executor even runs.
